# Referência da API aam-rs

`aam-rs` é a biblioteca oficial de Rust para analisar, validar e construir ficheiros de configuração AAM. Suporta diretivas (`@import`, `@derive`, `@schema`, `@type`), validação de tipos baseada em schema, pesquisa bidirecional e resolução de referência profunda.

A crate expõe duas APIs principais:
- **API `AAML`**: A estrutura core/legacy, amplamente usada e rica em funcionalidades.
- **API `AAM`**: Uma API nova de alto nível baseada em pipeline com suporte opcional de carga AOT (`.aam.bin`).

---

## Arquitetura

O projeto foi desenhado com uma arquitetura modular:

- **`aaml/`**: Lógica core para análise, pesquisa e validação.
- **`pipeline/`**: Um pipeline de cinco etapas (Lexer, Parser, Validator, Executer, Formatter).
- **`commands/`**: Sistema de diretivas (`@import`, `@derive`, etc.).
- **`types/`**: Sistema de tipos de validação para primitivos e tipos de domínio.
- **`aot/`**: Suporte para caches binárias "cozinhadas" (`.aam.bin`) para arranque rápido.
- **`builder/`**: API fluida para geração programática de conteúdo `.aam`.

---

## Feature Flags (Flags de Funcionalidades)

| Flag | Efeito |
|:---|:---|
| `hash-std` | Hasher por defeito do pipeline (`std::collections::hash_map::RandomState`) |
| `hash-fx` / `hash-ahash` / `hash-rapidhash` / `hash-ripemd` | Hashers alternativos (mutuamente exclusivos) |
| `perf-hash` | Altera o hasher de `HashMap` para `ahash::RandomState` para melhor throughput |
| `aot` | Habilita cache binária cozinhada em `src/aot/` e rota rápida `AAM::load` |
| `ffi` / `python` / `jni` / `csharp` | Habilita superfícies de bindings de linguagens |
| `parallel` | Habilita pré-validação paralela de tarefas no pipeline |
| `serde` | Deriva `Serialize`/`Deserialize` em `FoundValue`, `SchemaDef`, `AAMBuilder`, etc. |

## `AAML`

O ponto de entrada principal para todas as operações AAM. `AAML` mantém o mapa chave-valor analisado, definições de tipos registradas e definições de schema. Todas as diretivas (`@derive`, `@schema`, `@import`, `@type`) são processadas automaticamente no momento da análise.

### Construtores

#### `AAML::new`

Cria uma nova instância vazia de `AAML` com todos os manipuladores de diretivas por defeito registrados.

```rust
use aam_rs::aaml::AAML;

let cfg = AAML::new();
```

---

#### `AAML::parse`

Analisa uma string AAM e devolve uma nova instância de `AAML` totalmente preenchida.

| Parâmetro | Tipo | Descrição |
|:---|:---|:---|
| `content` | `&str` | Texto fonte AAM a analisar. |

**Retorno:** `Result<AAML, AamlError>`

---

#### `AAML::load`

Lê um ficheiro AAM do disco e devolve uma nova instância.

| Parámetro | Tipo | Descrição |
|:---|:---|:---|
| `file_path` | `impl AsRef<Path>` | Caminho para o ficheiro `.aam`. |

**Retorno:** `Result<AAML, AamlError>`

---

### Pesquisa (Lookup)

#### `find_obj`

O método principal de pesquisa. Procura `key` no mapa. Se não for encontrado, realiza uma **pesquisa inversa** automática por valor.

| Parâmetro | Tipo | Descrição |
|:---|:---|:---|
| `key` | `&str` | Clave ou valor a encontrar. |

**Retorno:** `Option<FoundValue>`

```rust
use aam_rs::aaml::AAML;

let cfg = AAML::parse("a = b").unwrap();

// Pesquisa direta
assert_eq!(cfg.find_obj("a").unwrap().as_str(), "b");

// Pesquisa inversa (procurando por valor)
assert_eq!(cfg.find_obj("b").unwrap().as_str(), "a");
```

---

## `FoundValue`

O tipo devolvido por todos os métodos de pesquisa. Envolve a string bruta e fornece ajudantes.

### Métodos

#### `as_str`

Devolve o valor interno como um slice de string.

**Retorno:** `&str`

---

#### `as_list`

Analisa o valor como um literal de lista AAM `[item, item, ...]` e devolve os elementos como `Vec<String>`.

**Retorno:** `Option<Vec<String>>`

---

## `AamlError`

Enum de erros usados ao longo dos caminhos de análise/validação/runtime.

### Variantes

| Variante | Descrição |
|:---|:---|
| `IoError(io::Error)` | Erro de I/O ao ler um ficheiro. |
| `ParseError { line, content, details }` | Uma linha não pôde ser analisada como instrução AAML válida. |
| `NotFound(String)` | Uma chave ou nome de tipo não foi encontrado. |
| `InvalidValue(String)` | Um valor falha uma restrição de tipo básica. |
| `InvalidType { type_name, details }` | Um valor falhou a validação contra um tipo. |
| `DirectiveError(String, String)` | Uma diretiva encontrou um erro nos seus argumentos. |
| `SchemaValidationError { ... }` | Uma restrição de schema foi violada. |

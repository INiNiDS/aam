# 类型检查

在解析阶段，高质量的加载器引擎必须确认**每个值都与其声明的类型兼容**。本页描述在最终确定 `AAML` 映射之前如何对每类AAM类型执行该检查。

---

## 概述

类型检查分两个阶段进行：

1. **解析** — 将 `@type` 别名展开为其规范类型（参见 [`@type`](../directives/type)）。
2. **检查** — 将解析的运行时节点与规范类型进行比较。

---

## 原始类型

原始类型是最简单的情况——检查 `PrimitiveTypes` 变量是否与预期的类型标签匹配：

| AAM类型 | `AamValue` 变量 | 接受的字面量示例 |
|---------|----------------|----------------|
| `i32` | `PrimitiveTypes::I32` | `42`, `-7`, `0` |
| `f64` | `PrimitiveTypes::F64` | `3.14`, `-0.5`, `1.0` |
| `string` | `PrimitiveTypes::Str` | `"hello"`, `""` |
| `bool` | `PrimitiveTypes::Bool` | `true`, `false` |
| `color` | `PrimitiveTypes::Color` | `#ff0000`, `#0a0b0c` |

:::tip 数值强制
AAM**不会**静默地将 `i32` 强制转换为 `f64` 或反之。`42` 始终是 `i32`；`42.0` 始终是 `f64`。不要添加隐式强制转换。
:::

---

## 验证清单

| 检查 | 必需 |
|------|------|
| 原始值与声明的原始类型匹配 | ✅ |
| 复合值 `type_key` 与预期单位匹配 | ✅ |
| `time::datetime` 解析为有效的ISO 8601 | ✅ |
| `list<T>` 的每个元素匹配 `T` | ✅ |
| `@type` 别名在比较前解析 | ✅ |
| 必填模式字段存在 | ✅ |
| 没有未知模式字段（严格模式） | ✅ |
| 嵌套模式对象递归验证 | ✅ |
| 没有隐式数值强制（`i32` ↔ `f64`） | ✅ |

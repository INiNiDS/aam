# 库架构

无论目标语言如何，每个现代AAM库都构建为**配置引擎**。遵循参考实现 `aam-rs` 的架构，引擎由三个主要子系统组成。理解它们的职责是正确实现的第一步。

```
┌─────────────┐     文本      ┌─────────────┐   AAML对象   ┌─────────────┐
│  .aam文件   │  ──────────►  │   加载器    │  ─────────► │  查询引擎   │
└─────────────┘               └──────┬──────┘             └─────────────┘
                                     │  ▲
                               AAML对象（合并后）
                                     │  │
                               ┌─────▼───────┐
                               │   构建器    │
                               └─────────────┘
                                     │
                                     │  文本
                                     ▼
                               ┌─────────────┐
                               │  .aam文件   │
                               └─────────────┘
```

---

## 1. 解析器和加载器

加载器是您的库的**入口点**。它读取原始AAM文本（或文件路径）并生成统一的内存表示——通常称为 `AAML` 或您语言中的等效结构体/映射。

### 职责

1. **标记化和解析** — 将源文本拆分为键值对、字符串和指令（`@import`、`@schema`、`@type`）。
2. **文件加载和合并** — 当遇到 `@import "other.aam"` 指令时，加载器无缝读取该文件、解析它并将其内容合并到当前文档中。
3. **数据模型构建** — 构建存储配置图的内部映射。

---

## 2. 查询引擎

查询引擎使AAM强大。与其直接公开原始映射，不如库提供智能遍历方法来读取配置。

### 职责

1. **双向查找（`find_obj`）** — 有时开发者知道键；有时他们知道值需要找到键。您的引擎应首先尝试直接键查找，如果失败则执行反向查找。
2. **递归解析（`find_deep`）** — AAM用户经常使用别名（例如 `current_path = base_path`）。查询引擎必须跟随值链直到找到不是键的终端值。
3. **循环检测** — 由于AAM是声明性的，用户可能很容易创建循环引用（例如 `a = b` 和 `b = a`）。`find_deep` 函数必须检测循环并返回最后访问的唯一值，以防止无限递归和堆栈溢出。
4. **抽象值包装（`FoundValue`）** — 不返回原始字符串，而是将结果包装在丰富的类型（如自定义类或结构体）中，允许轻松转换为整数、布尔值或就地字符串操作。

---

## 3. 配置构建器

构建器执行**反向**应用程序流：它允许用户以程序方式安全且高效地生成AAM文本，而无需手动连接字符串。

### 职责

1. **流畅的API生成** — 提供顺序添加键值对、注释、类型别名和 `@schema` 块的方法。
2. **确定性输出** — 构建器必须保持元素的精确插入顺序，以使生成的 `.aam` 文件保持干净且可差异比较。
3. **验证和格式化** — 确保带空格的字符串自动被引用，换行符正确放置，注释以 `#` 为前缀。

---

## 整合在一起

遵循配置引擎模型的典型库交互（翻译为Python）：

```python
import aam

# 1. 解析和加载（自动处理所有@import）
config = aam.load("config.aam")

# 2. 智能查询（深度解析）
terminal_path = config.find_deep("app_path")
print(f"解析的路径: {terminal_path}")

# 3. 以程序方式创建新配置
builder = aam.AAMBuilder()
builder.comment("生成的覆盖")
builder.add_line("app_path", terminal_path)
builder.to_file("overrides.aam")
```

通过解耦这三个核心系统，您的库为开发者提供绝对控制：他们可以解析任意配置、安全地解析复杂的依赖网络，或以程序方式生成健壮的文件。

# `@schema` 指令

`@schema` 指令**定义了一个命名数据契约**。它声明了文档必须（或可以）包含的字段，以及每个字段的类型。库必须解析这些定义并在验证期间使用它们。

```aam
@schema Player {
    name: string
    score: i32
    team*: string
}
```

---

## 语法

```
@schema <Name> {
    <field>:  <type>   # 必填字段
    <field>*: <type>   # 可选字段
}
```

主体可以写在一行（逗号分隔）或多行（换行分隔）：

```aam
# 多行（推荐用于可读性）
@schema Config {
    host: string
    port: i32
    debug*: bool
}

# 内联（适合小型模式）
@schema Point {x: f64, y: f64}
```

两种形式在语义上是相同的。您的解析器必须处理两者。

---

## 字段修饰符

| 语法 | 含义 |
|------|------|
| `name: type` | **必填** — 字段*必须*出现在使用此模式的任何文档中 |
| `name*: type` | **可选** — 字段*可以*省略；缺失不是错误 |

---

## 模式验证

解析完成后，验证器强制执行模式。有三个检查：

### 1 — 必填字段存在

对于 `required == true` 的每个字段，文档的 `entries` 必须包含匹配的键。

### 2 — 无未知字段

如果严格模式处于活动状态（推荐），则拒绝模式中未声明的任何键。

### 3 — 类型正确性

对于模式中*确实存在*的每个条目，验证值是否与声明的类型匹配（参见[类型检查](../validation/type-checking)）。

# `@import` 指令

`@import` 指令指示解析器将**外部 `.aam` 文件合并**到当前文档中。导入文件中的所有键值对、类型别名和模式定义在当前范围内变得可用。

```aam
@import path/to/other.aam
```

---

## 语法

`@import` 接受单个**引号字符串**——要导入的文件的路径：

```aam
@import "shared/defaults.aam"
@import "../common/colors.aam"
```

路径相对于**包含指令的文件**解析，而不是进程的工作目录。

---

## `@import` 的作用

当解析器遇到 `@import "file.aam"` 时：

1. 相对于当前文件的目录**解析路径**。
2. **规范化**它（解析 `..`、符号链接等）以获得循环检测的稳定标识。
3. 使用相同的解析器实例和相同的 `visited` 集合**递归解析导入的文件**。
4. 使用 **Child-Wins-on-Conflict** 语义将结果**合并**到当前文档中。

---

## 循环依赖保护

:::warning 防循环——此部分是实现的强制要求
如果没有循环检测，相互导入的两个文件将导致**无限递归**和堆栈溢出。此保护不是可选的。
:::

维护当前正在处理的**规范路径** 的 `HashSet<PathBuf>`（或您语言中的等效项）。处理任何导入之前：

1. 检查规范路径是否已在集合中。
2. **如果是** — 立即返回 `CyclicImport` 错误。
3. **如果否** — 添加它，递归，然后在递归调用返回后**删除它**。

---

## `@import` vs `@derive`

| 功能 | `@import` | `@derive` |
|------|-----------|-----------|
| 目标 | 整个文件 | 特定的 `@schema` |
| 模式强制 | 否 | 是 |
| 冲突解决 | Child-Wins | Child-Wins |
| 循环保护 | 必需 | 必需 |
| 典型用例 | 共享配置值 | 基于模式的数据契约 |

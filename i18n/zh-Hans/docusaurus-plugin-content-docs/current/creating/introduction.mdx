# AAM开发入门

本节帮助您使用首选编程语言为AAM创建自己的插件或库。

例如，如果您需要Rust的AAM，可以使用[官方 `aam-rs` 库](https://crates.io/crates/aaml)。如果您的语言（例如Python、Go、TypeScript）没有库，可以从头开始创建！`aam-rs` 实现被认为是AAM库设计的**黄金标准**，是健壮AAM库应如何构建的主要参考。

## 入门

首先，检查您的语言是否已存在库。
* **如果存在：** 贡献！添加新功能、修复错误或改善文档。
* **如果不存在：** 按照本节中的指南从头开始创建。

## 配置引擎范式

如果您正在构建库，需要了解AAM实际上是什么。**AAM是一种声明性文本配置格式**，建立在简单的键值对上，但通过文件导入、别名、递归深度解析和循环依赖安全等强大的结构化功能而出色。

从历史上看，AAM库被概念化为基本的"解析器、序列化器和验证器"集合。然而，以 `aam-rs` 为模型的现代AAM库作为综合**配置引擎**运作。成熟的高质量AAM库提供以下三个核心机制：

1. **解析器和加载器引擎：** 读取原始文本（`.aam` 文件）或字符串，通过遍历和将外部文件合并到统一的内存数据结构中无缝处理内联 `@import` 指令（类似于Python的 `dict`、Go的 `map` 或Rust的 `struct`）。
2. **智能查询引擎：** 允许用户安全高效地遍历配置图。这意味着提供双向查找（通过值查找键，或通过键查找值）和内置循环检测的递归"深层"别名解析，以防止堆栈溢出。
3. **配置构建器：** 为开发者提供程序化、流畅的API，通过代码动态地迭代构建、操作和保存 `.aam` 配置。

当您创作新库时，目标是将这些强大的机制转化为完全自然适合目标语言生态系统的惯用代码。

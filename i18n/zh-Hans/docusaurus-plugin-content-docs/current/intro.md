# AAM（抽象别名映射）
## AAM 介绍
配置文件往往会无节制地增长。它们变得臃肿、难以阅读，维护更是难上加难。将其拆分为多个文件有所帮助，但并不能解决管理数据关系这一本质复杂性。我们需要一种更直观的方式来处理映射——我们需要 AAM。
### 问题——冗长的映射
想象一下，您需要管理大量双向别名（可以通过值查找键，也可以通过键查找值）。在 TOML 等格式中，您不得不写大量样板代码：
```toml
[alias]
a = "b"
b = "a"
```
一个映射就要三行。乘以一百，您的配置文件就会变成噩梦。
### 解决方案——AAM
AAM 是一种简单、优雅的语法，让您可以用一行定义双向映射。使用 AAM，您可以写：
```aam
a = b
```
这一行创建了 `a` 和 `b` 之间的双向映射。查找 `a` 可得到 `b`，查找 `b` 可得到 `a`。简洁、清晰，消除了样板代码。
AAM 专为简洁、可读、易用而设计，去除样板代码，让您专注于配置逻辑。
## 灵感来源
我曾在一个需要管理大量别名的项目中工作。我使用 TOML 作为配置文件格式，发现管理别名非常困难。需要编写大量样板代码，而且很容易出错。我想要一种更好的方式来管理别名，于是便产生了 AAM 的想法。

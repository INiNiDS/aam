# はじめる
これはAAMのクイック入門です。基本をカバーし、2つの変数間のシンプルな双方向マッピングを作成する方法を示します。このガイドにはプロジェクトでAAMを使い始めるために必要なすべての情報が含まれています。

## MVP
AAMの**最小実行可能プロダクト**（MVP）はたった1行です：
```aam
a = b
```
この構文は`a`と`b`の双方向マッピングを作成します。`a`を検索すれば`b`が得られ、`b`を検索すれば`a`が得られます。簡潔で明瞭、定型文を排除します。
## 構文標準
AAMは可読性とシンプルさのために設計されたクリーンで最小限の構文に従っています。

### キーと値のマッピング
`=`演算子の両側に1つのスペースが必要です：
```aam
a = b
```

### 文の終端
文は**改行**（`\n`）で終端します。文を完結させるためにセミコロンやカンマは不要です：
```aam
a = b
c = d
e = f
```
*注意*: 文の終端にカンマは使いませんが、リスト (```["Hello", "World"]```) やインラインスキーマ (```{name: string, id: i32}```) などの特定のインライン構造内では要素の区切り文字として使われます。

### コメント
1行コメントは`#`プレフィックスを使います：
```aam
# これはコメントです
a = b
```

### ディレクティブ
ディレクティブ（インポートなどの宣言）は`@`プレフィックスを使います：
```aam
@import other-file.aam
```

### インラインスキーマ
インラインスキーマの区切り文字にはカンマ（`,`）を使います。例：
```aam
@schema Player {name: string, id: i32}
```

### 複数行スキーマ
複数行スキーマの区切り文字には改行（`\n`）を使います。例：
```aam
@schema Player {
    name: string
    id: i32
}
```

### 命名規則

プロジェクトを整潔で一貫性のある状態に保つため、AAMは異なる要素に特定の命名規則を使います：

* 変数とマッピング: すべての変数名とマッピングの代入にはsnake_caseを使います。
    ```aam
    player_score = current_points
    ```

* 組み込み型: i32、string、boolなどすべての基本型と標準型はsnake_caseで記述します。

* ユーザー定義型とスキーマ: 組み込み基本型と明確に区別するために、カスタム型とスキーマにはPascalCaseを使います。
    ```aam
    @schema PlayerData { ... } または @type UserProfile = string
    ```

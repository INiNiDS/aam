# スキーマ
AAMにおいてスキーマは構造化されたデータを定義する強力な方法です。明確な型と関係を持つ複雑なデータモデルを作成できます。スキーマはデータの形状の定義、入力の検証、コード生成など様々な目的に使えます。
## スキーマの定義
AAMでスキーマを定義するには、`@schema`ディレクティブの後にスキーマ名とフィールドおよびその型を含むブロックを記述します。例：
```aam
@schema Player {
    name: string
    id: i32
}
```
この例では、`name`（string型）と`id`（i32型）の2つのフィールドを持つ`Player`スキーマを定義します。カンマを区切り文字としてインラインでスキーマを定義することもできます：
```aam
@schema Player {name: string, id: i32}
```
どちらの定義も同等で、設定全体で使用できるスキーマを作成します。
## スキーマの使用
スキーマを定義したら、`@derive`ディレクティブを使ってそのスキーマのインスタンスを作成できます。例：
```aam
# file.aam
@schema Player {
    name: string
    id: i32
}
```
```aam
# main.aam
@derive file.aam::Player
name = "Alice"
id = 1
```
この例では`file.aam`から`Player`スキーマを継承し、`Player`スキーマのインスタンスの`name`と`id`フィールドを設定します。
## スキーマの必須フィールド
デフォルトでは、スキーマのすべてのフィールドが必須です。スキーマのインスタンスを作成する際はすべてのフィールドに値を指定する必要があります。必須フィールドの値を指定せずにインスタンスを作成しようとすると、AAMはエラーをスローします。例：
```aam
@derive file.aam::Player
name = "Alice"  # 'id'は必須ですが指定されていないためエラーになります
```
ただし、フィールド名の後に`*`を付けると省略可能になります：
```aam
@schema Player {
    name: string
    id*: i32
}
```
この場合、`id`フィールドは省略可能で、`id`の値なしで`Player`スキーマのインスタンスを作成できます：
```aam
@derive file.aam::Player
name = "Alice"  # 'id'は省略可能なので有効です
```
## チェック
### 型チェック
AAMはスキーマのインスタンスを作成する際にフィールドの型を自動チェックします。定義された型に一致しない値を割り当てようとすると、AAMはエラーをスローします。例：
```aam
@derive file.aam::SchemaName
name = 123  # エラー: 'name'はstringとして定義されています
id = "abc"  # エラー: 'id'はi32として定義されています
```
### 必須フィールドのチェック
AAMはスキーマのインスタンスを作成する際にすべての必須フィールドが指定されているかもチェックします。必須フィールドを省略するとAAMはエラーをスローします。例：
```aam
@derive file.aam::SchemaName
name = "Alice"  # エラー: 'id'は必須ですが指定されていません
```

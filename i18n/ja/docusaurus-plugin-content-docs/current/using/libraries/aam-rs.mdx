# aam-rs API リファレンス

`aam-rs` は、AAM 設定ファイルのパース、検証、および作成のための公式 Rust ライブラリです。ディレクティブ (`@import`, `@derive`, `@schema`, `@type`)、スキーマベースの型検証、双方向検索、および深い参照解決をサポートしています。

このクレートは2つの主要な API を提供します：
- **`AAML` API**: コア/レガシー構造体で、広く使用されており機能が豊富です。
- **`AAM` API**: パイプラインに基づいた新しい高レベル API で、オプションで AOT (`.aam.bin`) ロードをサポートしています。

---

## アーキテクチャ

このプロジェクトはモジュール化されたアーキテクチャで設計されています：

- **`aaml/`**: パース、検索、および検証のコアロジック。
- **`pipeline/`**: 5段階のパイプライン (Lexer, Parser, Validator, Executer, Formatter)。
- **`commands/`**: ディレクティブシステム (`@import`, `@derive` など)。
- **`types/`**: プリミティブおよびドメイン型の検証型システム。
- **`aot/`**: 高速起動のためのバイナリキャッシュ（調理済みファイル `.aam.bin`）のサポート。
- **`builder/`**: `.aam` コンテンツをプログラムで生成するための流暢な API。

---

## フィーチャーフラグ (Feature Flags)

| フラグ | 効果 |
|:---|:---|
| `hash-std` | デフォルトのパイプラインハッシャー (`std::collections::hash_map::RandomState`) |
| `hash-fx` / `hash-ahash` / `hash-rapidhash` / `hash-ripemd` | 代替ハッシャー（排他的） |
| `perf-hash` | `HashMap` のハッシャーを `ahash::RandomState` に変更し、スループットを向上させます |
| `aot` | `src/aot/` の調理済みバイナリキャッシュと `AAM::load` 高速パスを有効にします |
| `ffi` / `python` / `jni` / `csharp` | 言語バインディングを有効にします |
| `parallel` | パイプラインでのパースタスクの並列事前検証を有効にします |
| `serde` | `FoundValue`, `SchemaDef`, `AAMBuilder` などで `Serialize`/`Deserialize` を導出します |

## `AAML`

すべての AAM 操作の主要なエントリポイントです。 `AAML` は、パースされたキーと値のマップ、登録された型定義、およびスキーマ定義を保持します。すべてのディレクティブ (`@derive`, `@schema`, `@import`, `@type`) は、パース時に自動的に処理されます。

### コンストラクタ

#### `AAML::new`

デフォルトのディレクティブハンドラがすべて登録された、新しい空の `AAML` インスタンスを作成します。

```rust
use aam_rs::aaml::AAML;

let cfg = AAML::new();
```

---

#### `AAML::parse`

AAM 文字列をパースし、すべてのデータが入力された新しい `AAML` インスタンスを返します。

| パラメータ | 型 | 説明 |
|:---|:---|:---|
| `content` | `&str` | パースする AAM ソーステキスト。 |

**戻り値:** `Result<AAML, AamlError>`

---

#### `AAML::load`

ディスクから AAM ファイルを読み取り、新しいインスタンスを返します。

| パラメータ | 型 | 説明 |
|:---|:---|:---|
| `file_path` | `impl AsRef<Path>` | `.aam` ファイルへのパス。 |

**戻り値:** `Result<AAML, AamlError>`

---

### 検索 (Lookup)

#### `find_obj`

主要な検索メソッド。マップ内で `key` を探します。見つからない場合は、値による自動的な**逆引き検索**が実行されます。

| パラメータ | 型 | 説明 |
|:---|:---|:---|
| `key` | `&str` | 検索するキーまたは値。 |

**戻り値:** `Option<FoundValue>`

```rust
use aam_rs::aaml::AAML;

let cfg = AAML::parse("a = b").unwrap();

// 直接検索
assert_eq!(cfg.find_obj("a").unwrap().as_str(), "b");

// 逆引き検索 (値で検索)
assert_eq!(cfg.find_obj("b").unwrap().as_str(), "a");
```

---

## `FoundValue`

すべての `AAML` 検索メソッドによって返される型です。生の文字列値をラップし、便利なヘルパーを提供します。

### メソッド

#### `as_str`

内部の値を文字列スライスとして返します。

**戻り値:** `&str`

---

#### `as_list`

型を AAM リストリテラル `[item, item, ...]` としてパースし、要素を `Vec<String>` として返します。

**戻り値:** `Option<Vec<String>>`

---

## `AamlError`

パース、検証、および実行の各工程で使用されるエラー列挙型です。

### バリアント

| バリアント | 説明 |
|:---|:---|
| `IoError(io::Error)` | ファイル読み取り中の I/O エラー。 |
| `ParseError { line, content, details }` | 行が有効な AAML 命令としてパースできませんでした。 |
| `NotFound(String)` | キーまたは型名が見つかりませんでした。 |
| `InvalidValue(String)` | 値が基本的な型制約に違反しています。 |
| `InvalidType { type_name, details }` | 値が型に対する検証に失敗しました。 |
| `DirectiveError(String, String)` | ディレクティブの引数にエラーがありました。 |
| `SchemaValidationError { ... }` | スキーマ制約に違反しました。 |

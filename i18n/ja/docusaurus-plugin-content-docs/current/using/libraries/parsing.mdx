# パース (Parsing)

このページでは、公式の AAM ライブラリを使用して AAM コンテンツをアプリケーションにロードおよびパースするためのすべての方法を網羅しています。例では Rust や Python が多く使用されていますが、サポートされているすべての言語で同じ原則が適用されます。

---

## 構文とパーサーの動作

AAM は、特定のパースルールを持つ行ベースの `キー = 値` 形式を使用します：

### コメント

`#` 文字はコメントに使用されますが、**空白に囲まれている場合にのみ**有効です。これにより、16進数のカラー値（例： `tint = #ff6600`）をコメントと混同することなく正しくパースできます。

```aam
# これはコメントです
tint = #ff6600  # これはコメントではありません
name = #NoSpaceHere # これはコメントではありません（無効またはリテラル）
```

### 引用符

文字列値は、オプションで二重引用符 (`"`) または一重引用符 (`'`) で囲むことができます。必要に応じて、手動でこれらを取り除くために `unwrap_quotes` ユーティリティ（ほとんどのライブラリで利用可能）を使用できますが、多くの場合はパーサーによって自動的に処理されます。

---

## ファイルから

最も一般的なエントリーポイントは `AAML::load` で、ディスクから `.aam` ファイルを読み取り、すべてのディレクティブを処理し、すぐに使用できる `AAML` インスタンスを返します。

```rust
use aam_rs::aaml::AAML;

let cfg = AAML::load("config.aam").unwrap();
println!("{}", cfg.find_obj("host").unwrap());
```

```python
from aam_py import AAML

cfg = AAML.load("config.aam")
print(cfg.find_obj("host"))
```

## 文字列から

すでにメモリ内に文字列としてあるAAMコンテンツをパースするには `AAML::parse` を使用します。これはインライン設定、ユニットテスト、またはリモートソースから取得したコンテンツに便利です。

```rust
use aam_rs::aaml::AAML;

let source = "host = localhost\nport = 8080";
let cfg = AAML::parse(source).unwrap();

assert_eq!(cfg.find_obj("host").unwrap().as_str(), "localhost");
```

```python
from aam_py import AAML

source = "host = localhost\nport = 8080"
cfg = AAML.parse(source)

assert cfg.find_obj("host") == "localhost"
```

## 複数のソースのマージ

設定が複数のファイルや文字列から来る場合があります。`merge_content` と `merge_file` を使用して、既存の `AAML` インスタンスに追加のコンテンツを読み込みます。キーが既存のインスタンスとマージされるソースの両方に存在する場合、**受信値が勝ちます**。

```rust
use aam_rs::aaml::AAML;

let mut cfg = AAML::new();
cfg.merge_file("base.aam").unwrap();
cfg.merge_content("debug = true").unwrap(); // 'debug' を追加またはオーバーライド
```

```python
from aam_py import AAML

cfg = AAML()
cfg.merge_file("base.aam")
cfg.merge_content("debug = true")   # 'debug' を追加またはオーバーライド
```

## パース時に処理されるディレクティブ

すべてのAAMディレクティブはパース中に自動的に解決されます — `load` または `parse` 後に追加のメソッドを呼び出す必要はありません：

| ディレクティブ | 効果 |
|-------------|------|
| `@import` | 別の `.aam` ファイルをインラインで読み込みマージします。 |
| `@derive` | 別のファイルからキーと値のペアを継承します；キーの競合ではchildが勝ちます。 |
| `@schema` | 名前付きスキーマを登録し、フィールドごとの自動型チェックを有効にします。 |
| `@type` | スキーマフィールド宣言で使用するカスタム型エイリアスを登録します。 |

## エラーハンドリング

```rust
use aam_rs::aaml::AAML;
use aam_rs::error::AamlError;

match AAML::load("config.aam") {
    Ok(cfg) => {
        println!("host = {}", cfg.find_obj("host").unwrap());
    }
    Err(AamlError::IoError(e)) => {
        eprintln!("ファイルを読み取れませんでした: {e}");
    }
    Err(AamlError::ParseError { line, content, details }) => {
        eprintln!("{}行目でパースエラー: '{}' — {}", line, content, details);
    }
    Err(e) => eprintln!("予期しないエラー: {e}"),
}
```

```python
from aam_py import AAML, AamlError

try:
    cfg = AAML.load("config.aam")
    print("host =", cfg.find_obj("host"))
except AamlError as e:
    print(f"エラー: {e}")
```

# AAM開発入門

このカテゴリは、お好みのプログラミング言語でAAM用のプラグインまたはライブラリを作成するのに役立ちます。

たとえば、RustでAAMが必要な場合は、[公式 `aam-rs` ライブラリ](https://crates.io/crates/aaml)を使用できます。対象言語（例：Python、Go、TypeScript）のライブラリが存在しない場合は、ゼロから作成できます！ `aam-rs` の実装は、AAMライブラリ設計の**ゴールドスタンダード**とされており、堅牢なAAMライブラリがどのように構成されるべきかの主要なリファレンスです。

## はじめ方

まず、対象言語のライブラリがすでに存在するか確認してください。
* **存在する場合：** 貢献してください！新機能の追加、バグ修正、またはドキュメントの改善を行いましょう。
* **存在しない場合：** このセクションのガイドラインに従ってゼロから作成してください。

## 設定エンジンパラダイム

ライブラリを構築する前に、AAMが実際に何であるかを理解する必要があります。**AAMは宣言型テキスト設定フォーマット**であり、シンプルなキーと値のペアで構築されていますが、ファイルインポート、エイリアス、再帰的な深い解決、および循環依存の安全性といった強力な構造的機能によって優れています。

歴史的に、AAMライブラリは基本的な「パーサー、シリアライザー、バリデーター」のセットとして考えられていました。しかし、`aam-rs` をモデルにした現代のAAMライブラリは、包括的な**設定エンジン**として機能します。成熟した高品質のAAMライブラリは、以下の3つのコアメカニズムを提供します：

1. **パーサー＆ローダーエンジン：** 生のテキスト（`.aam` ファイル）または文字列を読み取り、外部ファイルをシームレスに処理してインメモリのデータ構造（Pythonの `dict`、Goの `map`、Rustの `struct` に相当）に統合します。
2. **スマートクエリエンジン：** 設定グラフを安全かつ効率的にトラバースできます。これは双方向ルックアップ（値でキーを見つける、またはキーで値を見つける）と、スタックオーバーフローを防ぐための組み込みループ検出による再帰的な「深い」エイリアス解決を提供します。
3. **設定ビルダー：** 開発者がコードを通じて `.aam` 設定を反復的に構築、操作、保存するためのプログラム的で流暢なAPIを提供します。

新しいライブラリを作成する際の目標は、これらの強力なメカニズムを対象言語のエコシステムに完全に自然なイディオマティックなコードに変換することです。

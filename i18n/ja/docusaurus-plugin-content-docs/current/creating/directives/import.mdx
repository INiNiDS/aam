# `@import` ディレクティブ

`@import` ディレクティブはパーサーに**外部の `.aam` ファイルを現在のドキュメントにマージ**するよう指示します。インポートされたファイルのすべてのキーと値のペア、型エイリアス、およびスキーマ定義が現在のスコープで利用可能になります。

```aam
@import path/to/other.aam
```

---

## 構文

`@import` は単一の**引用文字列**（インポートするファイルへのパス）を取ります：

```aam
@import "shared/defaults.aam"
@import "../common/colors.aam"
```

パスはプロセスの作業ディレクトリではなく、**ディレクティブを含むファイルに対して相対的に**解決されます。

---

## `@import` の動作

パーサーが `@import "file.aam"` を検出したとき：

1. 現在のファイルのディレクトリに対して相対的に**パスを解決**します。
2. サイクル検出のために安定したアイデンティティを得るために**正規化**します（`..`、シンボリックリンクなどを解決）。
3. 同じパーサーインスタンスと同じ `visited` セットを使用して**インポートされたファイルを再帰的にパース**します。
4. **Child-Wins-on-Conflict** セマンティクスを使用して結果を現在のドキュメントに**マージ**します — インポートするファイルにキーがすでに存在する場合、インポートするファイルの値が保持されます。

---

## `@import` vs `@derive`

| 機能 | `@import` | `@derive` |
|------|-----------|-----------|
| 対象 | ファイル全体 | 特定の `@schema` |
| スキーマの強制 | なし | あり |
| 競合解決 | Child-Wins | Child-Wins |
| サイクル保護 | 必須 | 必須 |
| 典型的なユースケース | 共有設定値 | スキーマベースのデータコントラクト |

# `@schema` ディレクティブ

`@schema` ディレクティブは**名前付きデータコントラクトを定義**します。ドキュメントが含まなければならない（または含む可能性のある）フィールドと各フィールドの型を宣言します。ライブラリはこれらの定義をパースし、後でバリデーション中に使用する必要があります。

```aam
@schema Player {
    name: string
    score: i32
    team*: string
}
```

---

## 構文

```
@schema <Name> {
    <field>:  <type>   # 必須フィールド
    <field>*: <type>   # 任意フィールド
}
```

本体は1行（カンマ区切り）または複数行（改行区切り）で書けます：

```aam
# 複数行（読みやすさのために推奨）
@schema Config {
    host: string
    port: i32
    debug*: bool
}

# インライン（小さいスキーマに便利）
@schema Point {x: f64, y: f64}
```

両方の形式は意味的に同一です。パーサーは両方を処理する必要があります。

---

## フィールド修飾子

| 構文 | 意味 |
|------|------|
| `name: type` | **必須** — フィールドはこのスキーマを使用するすべてのドキュメントに*必ず*存在する必要があります |
| `name*: type` | **任意** — フィールドは省略*できます*；不在はエラーではありません |

---

## スキーマバリデーション

パースが完了すると、バリデーターはスキーマを強制します。3つのチェックがあります：

### 1 — 必須フィールドの存在

`required == true` のすべてのフィールドに対して、ドキュメントの `entries` には一致するキーが含まれている必要があります。

### 2 — 不明なフィールドなし

厳格モードがアクティブな場合（推奨）、スキーマで宣言されていないキーを拒否します。

### 3 — 型の正確性

スキーマに*存在する*各エントリについて、値が宣言された型に一致することを確認します（[型チェック](../validation/type-checking) を参照）。

# `@type` ディレクティブ

`@type` ディレクティブを使用すると、ユーザーは既存の型（プリミティブ、複合型、または別のエイリアス）の**名前付きエイリアスを宣言**できます。ライブラリはパース中にこれらのエイリアスを登録し、その後の型参照（`@schema` フィールド、キーと値のアノテーションなど）が正規形式に解決できるようにする必要があります。

```aam
@type PlayerId = i32
@type SpawnPoint = types::math::vector3
@type Timeout = types::time::duration
```

---

## 構文

```
@type <Name> = <type_expression>
```

- `Name` は **PascalCase** でなければなりません（例：`UserId`、`WorldPosition`）。
- `type_expression` は任意の有効なAAM型 — プリミティブ、名前空間付き複合型、または以前に宣言されたエイリアス。
- 自己参照または前方参照エイリアスは**許可されません**；エイリアスはすでに登録された型のみを参照できます。

---

## スコープルール

型エイリアスはデフォルトで**ファイルスコープ**です。ファイルが `@import` を通じてインポートされると、その型エイリアスはインポートするドキュメントの `type_registry` にマージされます（キーと値のエントリと同様にChild-Wins-on-Conflictセマンティクスに従う）。

---

## 予約名

以下の名前はエイリアス名として**使用できません** — これらは組み込み型識別子です：

| 予約名 |
|--------|
| `i32`、`f64`、`string`、`bool`、`color` |
| `list` |
| `math::`、`physics::`、`time::` 名前空間の任意の名前 |

ユーザーが組み込みをシャドウしようとした場合、`AamlError::ReservedTypeName` エラーを返します。

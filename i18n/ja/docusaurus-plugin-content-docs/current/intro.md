# AAM (Abstract Alias Mapping)
## AAMについて
設定ファイルは気づかないうちに肥大化する傾向があります。ファイルは膨れ上がり、読みにくくなり、保守もさらに困難になります。複数のファイルに分割することは助けになりますが、データの関係を管理するという本質的な複雑さは解決しません。より直感的なマッピングの管理方法が必要でした — それがAAMです。
### 問題 — 冗長なマッピング
大量の双方向エイリアス（値からキーを、キーから値を検索できる）を管理する必要があると想像してください。TOMLのようなフォーマットでは、定型文を書かざるを得ません：
```toml
[alias]
a = "b"
b = "a"
```
1つのマッピングに3行かかります。これが100個あれば、設定ファイルは悪夢と化します。
### 解決策 — AAM
AAMは、双方向マッピングを1行で定義できるシンプルでエレガントな構文です。AAMを使えば：
```aam
a = b
```
この1行だけで`a`と`b`の双方向マッピングが作成されます。`a`を検索すれば`b`が得られ、`b`を検索すれば`a`が得られます。簡潔で明瞭、そして定型文を排除します。
AAMは簡潔さ・可読性・使いやすさのために設計されており、定型文を取り除くことで設定のロジックに集中できます。
## アイデアの誕生
大量のエイリアスを管理する必要があるプロジェクトに取り組んでいました。設定ファイルにTOMLを使っていましたが、エイリアスの管理が非常に困難でした。定型コードを大量に書く必要があり、ミスも起きやすかったです。より良い方法を求めた結果、AAMのアイデアが生まれました。

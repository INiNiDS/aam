# Referencia de la API aam-rs

`aam-rs` es la librería oficial de Rust para analizar, validar y construir ficheros de configuración AAM. Soporta directivas (`@import`, `@derive`, `@schema`, `@type`), validación de tipos basada en schema, búsqueda bidireccional y resolución de referencia profunda.

La crate expone dos APIs principales:
- **API `AAML`**: La estructura core/legacy, ampliamente usada y rica en funcionalidades.
- **API `AAM`**: Una API nueva de alto nivel basada en pipeline con soporte opcional de carga AOT (`.aam.bin`).

---

## Arquitectura

El proyecto está diseñado con una arquitectura modular:

- **`aaml/`**: Lógica core para análisis, búsqueda y validación.
- **`pipeline/`**: Un pipeline de cinco etapas (Lexer, Parser, Validator, Executer, Formatter).
- **`commands/`**: Sistema de directivas (`@import`, `@derive`, etc.).
- **`types/`**: Sistema de tipos de validación para primitivos y tipos de dominio.
- **`aot/`**: Soporte para cachés binarias "cocinadas" (`.aam.bin`) para inicio rápido.
- **`builder/`**: API fluida para generación programática de contenido `.aam`.

---

## Feature Flags (Flags de Funcionalidades)

| Flag | Efecto |
|:---|:---|
| `hash-std` | Hasher por defecto del pipeline (`std::collections::hash_map::RandomState`) |
| `hash-fx` / `hash-ahash` / `hash-rapidhash` / `hash-ripemd` | Hashers alternativos (mutuamente excluyentes) |
| `perf-hash` | Cambia el hasher de `HashMap` a `ahash::RandomState` para mejor throughput |
| `aot` | Habilita caché binaria cocinada en `src/aot/` y ruta rápida `AAM::load` |
| `ffi` / `python` / `jni` / `csharp` | Habilita superficies de bindings de lenguajes |
| `parallel` | Habilita pre-validación paralela de tareas en el pipeline |
| `serde` | Deriva `Serialize`/`Deserialize` en `FoundValue`, `SchemaDef`, `AAMBuilder`, etc. |

## `AAML`

El punto de entrada principal para todas las operaciones AAM. `AAML` mantiene el mapa clave-valor analizado, definiciones de tipos registradas y definiciones de schema. Todas las directivas (`@derive`, `@schema`, `@import`, `@type`) se procesan automáticamente en el momento del análisis.

### Constructores

#### `AAML::new`

Crea una nueva instancia vacía de `AAML` con todos los manejadores de directivas por defecto registrados.

```rust
use aam_rs::aaml::AAML;

let cfg = AAML::new();
```

---

#### `AAML::parse`

Analiza una string AAM y devuelve una nueva instancia de `AAML` totalmente poblada.

| Parámetro | Tipo | Descripción |
|:---|:---|:---|
| `content` | `&str` | Texto fuente AAM a analizar. |

**Retorno:** `Result<AAML, AamlError>`

---

#### `AAML::load`

Lee un fichero AAM del disco y devuelve una nueva instancia.

| Parámetro | Tipo | Descripción |
|:---|:---|:---|
| `file_path` | `impl AsRef<Path>` | Ruta al fichero `.aam`. |

**Retorno:** `Result<AAML, AamlError>`

---

### Búsqueda (Lookup)

#### `find_obj`

El método principal de búsqueda. Busca `key` en el mapa. Si no se encuentra, realiza una **búsqueda inversa** automática por valor.

| Parámetro | Tipo | Descripción |
|:---|:---|:---|
| `key` | `&str` | Clave o valor a encontrar. |

**Retorno:** `Option<FoundValue>`

```rust
use aam_rs::aaml::AAML;

let cfg = AAML::parse("a = b").unwrap();

// Búsqueda directa
assert_eq!(cfg.find_obj("a").unwrap().as_str(), "b");

// Búsqueda inversa (buscando por valor)
assert_eq!(cfg.find_obj("b").unwrap().as_str(), "a");
```

---

## `FoundValue`

El tipo devuelto por todos los métodos de búsqueda. Envuelve la string cruda y proporciona ayudantes.

### Métodos

#### `as_str`

Devuelve el valor interno como un slice de string.

**Retorno:** `&str`

---

#### `as_list`

Analiza el valor como un literal de lista AAM `[item, item, ...]` y devuelve los elementos como `Vec<String>`.

**Retorno:** `Option<Vec<String>>`

---

## `AamlError`

Enum de errores usados a lo largo de los caminos de análisis/validación/runtime.

### Variantes

| Variante | Descripción |
|:---|:---|
| `IoError(io::Error)` | Error de I/O al leer un fichero. |
| `ParseError { line, content, details }` | Una línea no pudo ser analizada como sentencia AAML válida. |
| `NotFound(String)` | Una clave o nombre de tipo no se encontró. |
| `InvalidValue(String)` | Un valor falla una restricción de tipo básica. |
| `InvalidType { type_name, details }` | Un valor falló la validación contra un tipo. |
| `DirectiveError(String, String)` | Una directiva encontró un error en sus argumentos. |
| `SchemaValidationError { ... }` | Se violó una restricción de schema. |

# Référence de l'API aam-rs

`aam-rs` est la bibliothèque officielle Rust pour analyser, valider et construire des fichiers de configuration AAM. Elle supporte les directives (`@import`, `@derive`, `@schema`, `@type`), la validation de types basée sur schéma, la recherche bidirectionnelle et la résolution de référence profonde.

La crate expose deux API principales :
- **API `AAML`** : La structure core/legacy, largement utilisée et riche en fonctionnalités.
- **API `AAM`** : Une nouvelle API de haut niveau basée sur pipeline avec support optionnel de chargement AOT (`.aam.bin`).

---

## Architecture

Le projet est conçu avec une architecture modulaire :

- **`aaml/`** : Logique core pour l'analyse, la recherche et la validation.
- **`pipeline/`** : Un pipeline à cinq étapes (Lexer, Parser, Validator, Executer, Formatter).
- **`commands/`** : Système de directives (`@import`, `@derive`, etc.).
- **`types/`** : Système de types de validation pour les types primitifs et de domaine.
- **`aot/`** : Support pour les caches binaires "cuits" (`.aam.bin`) pour un démarrage rapide.
- **`builder/`** : API fluide pour la génération programmatique de contenu `.aam`.

---

## Feature Flags (Drapeaux de fonctionnalités)

| Flag | Effet |
|:---|:---|
| `hash-std` | Hasheur par défaut du pipeline (`std::collections::hash_map::RandomState`) |
| `hash-fx` / `hash-ahash` / `hash-rapidhash` / `hash-ripemd` | Hasheurs alternatifs (mutuellement exclusifs) |
| `perf-hash` | Remplace le hasheur `HashMap` par `ahash::RandomState` pour un meilleur débit |
| `aot` | Active le cache binaire cuit dans `src/aot/` et la route rapide `AAM::load` |
| `ffi` / `python` / `jni` / `csharp` | Active les surfaces de bindings de langages |
| `parallel` | Active la pré-validation parallèle des tâches dans le pipeline |
| `serde` | Dérive `Serialize`/`Deserialize` sur `FoundValue`, `SchemaDef`, `AAMBuilder`, etc. |

## `AAML`

Le point d'entrée principal pour toutes les opérations AAM. `AAML` maintient la map clé-valeur analysée, les définitions de types enregistrées et les définitions de schéma. Toutes les directives (`@derive`, `@schema`, `@import`, `@type`) sont traitées automatiquement au moment de l'analyse.

### Constructeurs

#### `AAML::new`

Crée une nouvelle instance vide de `AAML` avec tous les gestionnaires de directives par défaut enregistrés.

```rust
use aam_rs::aaml::AAML;

let cfg = AAML::new();
```

---

#### `AAML::parse`

Analyse une chaîne AAM et retourne une nouvelle instance `AAML` entièrement remplie.

| Paramètre | Type | Description |
|:---|:---|:---|
| `content` | `&str` | Texte source AAM à analyser. |

**Retour :** `Result<AAML, AamlError>`

---

#### `AAML::load`

Lit un fichier AAM du disque et retourne une nouvelle instance.

| Paramètre | Type | Description |
|:---|:---|:---|
| `file_path` | `impl AsRef<Path>` | Chemin vers le fichier `.aam`. |

**Retour :** `Result<AAML, AamlError>`

---

### Recherche (Lookup)

#### `find_obj`

La méthode de recherche principale. Cherche `key` dans la map. Si non trouvée, effectue une **recherche inverse** automatique par valeur.

| Paramètre | Type | Description |
|:---|:---|:---|
| `key` | `&str` | Clé ou valeur à trouver. |

**Retour :** `Option<FoundValue>`

```rust
use aam_rs::aaml::AAML;

let cfg = AAML::parse("a = b").unwrap();

// Recherche directe
assert_eq!(cfg.find_obj("a").unwrap().as_str(), "b");

// Recherche inverse (recherche par valeur)
assert_eq!(cfg.find_obj("b").unwrap().as_str(), "a");
```

---

## `FoundValue`

Le type retourné par toutes les méthodes de recherche. Enveloppe la chaîne brute et fournit des aides.

### Méthodes

#### `as_str`

Retourne la valeur interne sous forme de slice de chaîne.

**Retour :** `&str`

---

#### `as_list`

Analyse la valeur comme un littéral de liste AAM `[item, item, ...]` et retourne les éléments sous forme de `Vec<String>`.

**Retour :** `Option<Vec<String>>`

---

## `AamlError`

Enum d'erreurs utilisées tout au long des chemins d'analyse/validation/runtime.

### Variantes

| Variante | Description |
|:---|:---|
| `IoError(io::Error)` | Erreur I/O lors de la lecture d'un fichier. |
| `ParseError { line, content, details }` | Une ligne n'a pas pu être analysée comme instruction AAML valide. |
| `NotFound(String)` | Une clé ou un nom de type n'a pas été trouvé. |
| `InvalidValue(String)` | Une valeur échoue à une contrainte de type basique. |
| `InvalidType { type_name, details }` | Une valeur a échoué la validation contre un type. |
| `DirectiveError(String, String)` | Une directive a rencontré une erreur dans ses arguments. |
| `SchemaValidationError { ... }` | Une contrainte de schéma a été violée. |

# Директива `@type`

Директива `@type` позволяет пользователю **объявить именованный псевдоним** для любого существующего типа — примитивного, сложного или другого псевдонима. Библиотеки должны регистрировать эти псевдонимы во время разбора, чтобы любая последующая ссылка на тип (в полях `@schema`, аннотациях ключ-значение и т. д.) могла быть разрешена до канонической формы.

```aam
@type PlayerId = i32
@type SpawnPoint = types::math::vector3
@type Timeout = types::time::duration
```

---

## Синтаксис

```
@type <Name> = <type_expression>
```

- `Name` должен быть в **PascalCase** (например, `UserId`, `WorldPosition`).
- `type_expression` — любой допустимый тип AAM: примитив, пространство имён комплексного типа или ранее объявленный псевдоним.
- Самоссылающиеся или форвард-ссылочные псевдонимы **не допускаются**; псевдонимы должны ссылаться только на уже зарегистрированные типы.

---

## Что делает `@type`

Когда парсер встречает `@type Alias = SomeType`:

1. Разбирает `SomeType` как выражение типа.
2. Разрешает `SomeType` до канонического `AamType` с использованием текущего `type_registry`.
3. Регистрирует отображение `Alias → canonical AamType` в `type_registry`.
4. С этого момента `Alias` может использоваться везде, где ожидается тип.

---

## Внутреннее представление

Реестр типов — это плоская `HashMap<String, AamType>`, хранящаяся в `AamDocument`:

```rust
pub enum AamType {
    // Примитивы
    I32,
    F64,
    String,
    Bool,
    Color,

    // Комплексные типы с пространством имён
    Physics(PhysicsType),
    Math(MathType),
    Time(TimeType),

    // Составные
    List(Box<AamType>),
    Schema(String),   // ссылается на именованный @schema

    // Разрешённый псевдоним — прозрачная обёртка
    Alias { name: String, resolved: Box<AamType> },
}

// В AAML:
pub type_registry: HashMap<String, String>,  // или эквивалентные представления
```

Когда потребитель спрашивает «каков тип поля `x`?», движок запросов должен **разворачивать псевдонимы** до тех пор, пока не достигнет конкретного варианта базового типа.

---

## Разбор объявления `@type`

```rust
fn parse_type_directive(
    tokens: &mut TokenStream,
    doc: &mut AAML,
) -> Result<(), AamlError> {
    let alias_name = tokens.expect_identifier()?; // например, "PlayerId"
    tokens.expect(Token::Equals)?;

    // Разрешаем правую часть относительно текущего реестра
    let canonical = parse_type_expression(tokens, &doc.type_registry)?;

    // Защита: имя псевдонима не должно совпадать со встроенным типом
    if is_builtin_type(&alias_name) {
        return Err(AamlError::ReservedTypeName { name: alias_name });
    }

    doc.type_registry.insert(
        alias_name.clone(),
        canonical // Сохраняем разрешённое строковое представление
    );

    Ok(())
}
```

## Цепочки псевдонимов

Псевдонимы могут быть вложенными — один псевдоним может ссылаться на другой:

```aam
@type Name = string
@type FirstName = Name
@type LastName = Name
```

Ваша функция `parse_type_expression` должна рекурсивно выполнять поиск в реестре до тех пор, пока не достигнет не-псевдонимного типа. Реализуйте вспомогательную функцию для канонизации:

```rust
fn resolve_type(ty: &AamType, registry: &HashMap<String, AamType>) -> AamType {
    match ty {
        AamType::Alias { resolved, .. } => resolve_type(resolved, registry),
        other => other.clone(),
    }
}
```

Используйте `resolve_type` в валидаторе, чтобы сравнение типов всегда выполнялось для канонических типов, а не для псевдонимных обёрток.

### Защита от циклических псевдонимов

Хотя форвард-ссылки уже запрещены (псевдоним может ссылаться только на уже зарегистрированные типы), всё же следует защититься от патологических случаев:

```rust
fn resolve_type_bounded(
    ty: &String,
    registry: &HashMap<String, String>,
    depth: u8,
) -> Result<String, AamlError> {
    if depth > 32 {
        return Err(AamlError::TypeAliasCycle);
    }
    // Выполняем логику проверки здесь, используя строки или варианты структур...
    Ok(ty.clone())
}
```

## Правила области видимости

Псевдонимы типов имеют **файловую область видимости по умолчанию**. Когда файл импортируется через `@import`, его псевдонимы типов сливаются в `type_registry` импортирующего документа (с семантикой «победы дочернего элемента при конфликте», как и для пар ключ-значение).

```aam
# types.aam
@type Velocity = physics::meterpersecond
@type Mass  = physics::kilogram
```

```aam
# simulation.aam
@import "types.aam"

# Velocity и Mass теперь доступны здесь
@schema Body {
    v: Velocity
    m: Mass
}
```

## Использование псевдонимов в схемах

После регистрации псевдоним неотличим от встроенного типа в месте использования:

```aam
@type EntityId = i32

@schema Entity {
    id: EntityId      # разрешается в i32
    label: string
}
```

Валидатор должен разрешить `EntityId → i32` перед проверкой значения поля `id`, чтобы `EntityId` и `i32` рассматривались как эквивалентные.

---

## Зарезервированные имена

Следующие имена **нельзя** использовать в качестве псевдонимов — они являются встроенными идентификаторами типов:

| Зарезервированные имена |
|------------------------|
| `i32`, `f64`, `string`, `bool`, `color` |
| `list` |
| Любое имя в пространствах имён `math::`, `physics::`, `time::` |

Возвращайте ошибку `AamlError::ReservedTypeName`, если пользователь пытается переопределить встроенный тип.

# Введение в разработку AAM

Этот раздел поможет вам создать собственный плагин или библиотеку для AAM на удобном вам языке программирования.

Например, если вам нужен AAM для Rust, можно воспользоваться [официальной библиотекой `aam-rs`](https://crates.io/crates/aaml). Если библиотеки для вашего языка не существует (например, Python, Go, TypeScript), вы можете создать её с нуля! Реализация `aam-rs` считается **эталонной** для проектирования AAM-библиотек и служит основным источником информации о том, как должна быть устроена надёжная AAM-библиотека.

## Начало работы

Прежде всего проверьте, существует ли уже библиотека для вашего языка.
* **Если да:** Вносите вклад! Добавляйте новые возможности, исправляйте ошибки или улучшайте документацию.
* **Если нет:** Создайте её с нуля, следуя руководящим принципам этого раздела.

## Парадигма конфигурационного движка

Перед созданием библиотеки нужно понять, чем является AAM на самом деле. **AAM — декларативный текстовый формат конфигурации**, построенный на простых парах «ключ—значение», но расширенный мощными структурными возможностями: импортом файлов, псевдонимами, рекурсивным глубоким разрешением значений и защитой от циклических зависимостей.

Исторически AAM-библиотеки воспринимались как наборы «парсеров, сериализаторов и валидаторов». Однако современные библиотеки, созданные по образцу `aam-rs`, действуют как полноценные **конфигурационные движки**. Зрелая высококачественная AAM-библиотека предоставляет три ключевых механизма:

1. **Движок разбора и загрузки:** Чтение необработанного текста (файлов `.aam`) или строк и прозрачная обработка встроенных директив `@import` путём обхода и слияния внешних файлов в единую структуру в памяти (аналог `dict` в Python, `map` в Go или `struct` в Rust).
2. **Движок умных запросов:** Безопасный и эффективный обход графа конфигурации. Это означает поддержку двунаправленного поиска (нахождение ключей по значению и значений по ключу) и рекурсивного «глубокого» разрешения псевдонимов со встроенным обнаружением циклов для предотвращения переполнения стека.
3. **Конструктор конфигураций:** Предоставление разработчикам программного гибкого API для итеративного создания, изменения и сохранения конфигураций `.aam` динамически через код.

При разработке новой библиотеки ваша цель — реализовать эти мощные механизмы в виде идиоматического кода, естественного для экосистемы целевого языка.

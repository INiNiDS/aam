# Проверка обязательных ключей
Необходимо подтвердить, что **все обязательные ключи присутствуют** в разобранной карте AAML перед её финализацией. Эта страница описывает, как выполнить такую проверку и корректно обработать любые отсутствующие ключи.
Обязательные поля определяются через `:`, например ```a: b```. Этот синтаксис указывает, что ключ `a` обязателен и должен иметь значение типа `b`. Если обязательный ключ отсутствует в разобранной карте AAML, следует вернуть ошибку, указывающую, какой ключ отсутствует и какой тип ожидался.

```rust
fn check_required_keys(
    parsed_map: &HashMap<String, AamValue>,
    schema: &Schema, // Содержит определения полей и флаги обязательности
) -> Result<(), AamlError> {
    for (field_name, field_def) in &schema.fields {
        if field_def.required && !parsed_map.contains_key(field_name) {
            return Err(AamlError::MissingRequiredField {
                field: field_name.clone(),
                schema: schema.name.clone(),
            });
        }
    }
    Ok(())
}
```
Необязательные поля определяются через `*:`, например ```a*: b```. Этот синтаксис указывает, что ключ `a` необязателен и может иметь значение типа `b`. Если необязательный ключ отсутствует в разобранной карте AAML, его можно просто игнорировать без возврата ошибки.
Проверка обязательных ключей должна выполняться после разбора документа AAML и до финализации карты `AAML`. Это гарантирует раннее обнаружение любых отсутствующих обязательных ключей в процессе загрузки и предоставляет пользователю чёткую обратную связь о том, что отсутствует.
Проверка должна выполняться в файле, наследующем схему (производным файле).

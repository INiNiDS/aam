# AAM (Abstract Alias Mapping)
## Знакомство с AAM
Конфигурационные файлы имеют тенденцию бесконтрольно разрастаться. Они становятся раздутыми, трудночитаемыми и ещё более трудными в обслуживании. Разбивка на несколько файлов помогает, но не решает основную проблему управления связями между данными. Нужен был более интуитивный способ работы с маппингами — нужен AAM.
### Проблема — избыточный маппинг
Представьте, что вам нужно управлять большим набором двунаправленных псевдонимов (где можно искать ключ по значению и наоборот). В форматах вроде TOML вы вынуждены писать шаблонный код:
```toml
[alias]
a = "b"
b = "a"
```
Три строки ради одного маппинга. Умножьте на сотню — и ваш конфиг превратится в кошмар.
### Решение — AAM
AAM — простой и элегантный синтаксис, позволяющий определить двунаправленный маппинг в одну строку. С AAM можно написать:
```aam
a = b
```
Эта единственная строка создаёт двунаправленный маппинг между `a` и `b`. По `a` можно получить `b`, а по `b` — `a`. Лаконично, понятно и без лишнего шаблонного кода.
AAM разработан для краткости, читаемости и простоты использования — он убирает весь шаблонный код, позволяя сосредоточиться на логике вашей конфигурации.
## Как родилась идея
Я работал над проектом, в котором нужно было управлять большим количеством псевдонимов. Я использовал TOML для конфигурационных файлов и обнаружил, что управлять псевдонимами крайне неудобно: приходилось писать много шаблонного кода, в котором легко ошибиться. Я хотел найти лучший способ — так и родилась идея AAM.

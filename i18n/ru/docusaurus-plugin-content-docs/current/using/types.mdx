# Типы

## Примитивные типы
AAM имеет несколько встроенных примитивных типов: `string`, `i32`, `f64`, `bool` и `color` (принимает hex-коды цветов, например `#ff0000` или `#00ff00`).

```aam
@schema User {
    name: string
    age: i32
    is_admin: bool
    profile_color: color
}
```

## Другие встроенные типы
AAM также предоставляет расширенные встроенные типы для специфических случаев использования.

### Физические типы

Для доступа к типам, связанным с физикой, необходимо использовать пространство имён `types::physics`. Например:

```aam
@schema PhysicsObject {
    length: types::physics::meter
}
```

Полный список физических типов можно найти в [документации по дополнительным типам](../types/physics).
### Математические типы
Для получения математических типов используйте пространство имён `types::math`, например:
```aam
@schema MathObjects {
    vector: types::math::vector2
}
```
Все математические типы описаны в [документации по дополнительным типам](../types/math).
### Время
AAM поддерживает типы для работы со временем. Для datetime и duration используется формат ISO 8601:
```aam
@schema TimeObjects {
    timestamp: types::time::datetime # например, "2024-01-01T12:00:00Z"
    cooldown: types::time::duration # например, "PT1H30M" для 1 часа 30 минут
    units: types::time::hour
}
```
Полный список доступных типов времени см. в [документации по дополнительным типам](../types/time).
### Списки
AAM поддерживает тип «список», позволяющий определить набор элементов определённого типа. Например:
```aam
@schema Inventory {
    items: list<string>
}
```
и создать список:
```aam
items = ["Hello", "World"]
```
## Собственные типы
### Схемы
Вы можете использовать собственные схемы как типы. Например:
```aam
@schema User {
    name: string
    age: i32
}

@schema ServerUsers {
    users: list<User>
}
```
### Пользовательские типы
Также можно объявить собственный тип с помощью директивы `@type`:
```aam
@type Name = string
```
Это позволяет создавать пользовательские типы, которые можно переиспользовать во всей конфигурации, улучшая читаемость и сопровождаемость.
Также можно создавать новые типы на основе непримитивных типов, например:
```aam
@type FromLast = types::time::duration
```
Или использовать собственные типы в других объявлениях типов:
```aam
@type Name = string
@type FirstName = Name
@type LastName = Name
```

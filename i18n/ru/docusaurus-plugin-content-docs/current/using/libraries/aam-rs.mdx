# Справочник API aam-rs

`aam-rs` — это официальная библиотека Rust для разбора, валидации и создания файлов конфигурации AAM. Она поддерживает директивы (`@import`, `@derive`, `@schema`, `@type`), валидацию типов на основе схем, двунаправленный поиск и глубокое разрешение ссылок.

Crate предоставляет два основных API:
- **API `AAML`**: Основная/устаревшая структура, широко используемая и многофункциональная.
- **API `AAM`**: Новый высокоуровневый API на базе пайплайна с дополнительной поддержкой загрузки AOT (`.aam.bin`).

---

## Архитектура

Проект имеет модульную архитектуру:

- **`aaml/`**: Основная логика разбора, поиска и валидации.
- **`pipeline/`**: Пятиэтапный конвейер (Lexer, Parser, Validator, Executer, Formatter).
- **`commands/`**: Система директив (`@import`, `@derive` и др.).
- **`types/`**: Система типов валидации для примитивов и доменных типов.
- **`aot/`**: Поддержка бинарных кэшей ("приготовленных" файлов `.aam.bin`) для быстрого запуска.
- **`builder/`**: Fluent-интерфейс для программной генерации содержимого `.aam`.

## Флаги возможностей (Feature Flags)

| Флаг | Эффект |
|:---|:---|
| `hash-std` | Стандартный хешер конвейера (`std::collections::hash_map::RandomState`) |
| `hash-fx` / `hash-ahash` / `hash-rapidhash` / `hash-ripemd` | Альтернативные хешеры (взаимоисключающие) |
| `perf-hash` | Заменяет хешер `HashMap` на `ahash::RandomState` для лучшей пропускной способности |
| `aot` | Включает приготовленный бинарный кэш `src/aot/` и быстрый путь `AAM::load` |
| `ffi` / `python` / `jni` / `csharp` | Включает интерфейсы привязок к языкам |
| `parallel` | Включает параллельную предварительную валидацию задач разбора в конвейере |
| `serde` | Добавляет реализацию `Serialize`/`Deserialize` для `FoundValue`, `SchemaDef`, `AAMBuilder` и др. |

## `AAML`

Основная точка входа для всех операций AAM. `AAML` хранит отпарсенную карту ключ-значение, зарегистрированные определения типов и определения схем. Все директивы (`@derive`, `@schema`, `@import`, `@type`) обрабатываются автоматически во время разбора.

### Конструкторы

#### `AAML::new`

Создает новый пустой экземпляр `AAML` со всеми зарегистрированными по умолчанию обработчиками директив (`@import`, `@derive`, `@schema`, `@type`).

```rust
use aam_rs::aaml::AAML;

let cfg = AAML::new();
```

---

#### `AAML::parse`

Разбирает строку AAM и возвращает новый, полностью заполненный экземпляр `AAML`. Все найденные в строке директивы обрабатываются до завершения метода.

| Параметр | Тип | Описание |
|:---|:---|:---|
| `content` | `&str` | Исходный текст AAM для разбора. |

**Возвращает:** `Result<AAML, AamlError>`

---

#### `AAML::load`

Читает файл AAM с диска и возвращает новый экземпляр `AAML`. Эквивалентно ручному чтению файла и вызову `AAML::parse`.

| Параметр | Тип | Описание |
|:---|:---|:---|
| `file_path` | `impl AsRef<Path>` | Путь к файлу `.aam`. |

**Возвращает:** `Result<AAML, AamlError>`

---

### Слияние

Эти методы добавляют содержимое в **существующий** экземпляр `AAML`, а не создают новый. Они полезны для компоновки конфигураций из нескольких источников во время выполнения.

#### `merge_content`

Разбирает исходный текст AAM и объединяет полученные пары ключ-значение и определения схем с текущим экземпляром. Существующие ключи перезаписываются входящими значениями.

| Параметр | Тип | Описание |
|:---|:---|:---|
| `content` | `&str` | Исходный текст AAM для разбора. |

**Возвращает:** `Result<(), AamlError>`

---

#### `merge_file`

Читает файл с диска и объединяет его содержимое с текущим экземпляром. Эквивалентно чтению файла и вызову `merge_content`.

| Параметр | Тип | Описание |
|:---|:---|:---|
| `file_path` | `impl AsRef<Path>` | Путь к файлу `.aam`. |

**Возвращает:** `Result<(), AamlError>`

---

### Поиск (Lookup)

Эти методы извлекают значения из отпарсенной карты ключ-значение. Все они возвращают `FoundValue` при успехе.

#### `find_obj`

Основной метод поиска. Ищет `key` в карте. Если совпадающий ключ не найден, выполняется автоматический **обратный поиск** — поиск любой записи, чье *значение* равно `key`.

| Параметр | Тип | Описание |
|:---|:---|:---|
| `key` | `&str` | Ключ или значение для поиска. |

**Возвращает:** `Option<FoundValue>`

```rust
use aam_rs::aaml::AAML;

let cfg = AAML::parse("a = b").unwrap();

// Прямой поиск
assert_eq!(cfg.find_obj("a").unwrap().as_str(), "b");

// Обратный поиск (по значению)
assert_eq!(cfg.find_obj("b").unwrap().as_str(), "a");
```

---

#### `find_key`

Явный обратный поиск. Находит **ключ**, соответствующее значение которого равно `value`.

| Параметр | Тип | Описание |
|:---|:---|:---|
| `value` | `&str` | Значение для поиска. |

**Возвращает:** `Option<FoundValue>` — ключ как `FoundValue` или `None`.

---

#### `find_deep`

Следует по цепочке поисков `ключ → значение → ключ`, пока не будет достигнуто конечное значение или не будет обнаружен цикл. Полезно для обхода связанных сопоставлений.

| Параметр | Тип | Описание |
|:---|:---|:---|
| `key` | `&str` | Начальный ключ для следования. |

**Возвращает:** `Option<FoundValue>` — итоговое конечное значение или `None`.

---

## `FoundValue`

Тип, возвращаемый всеми методами поиска `AAML`. Он оборачивает исходное строковое значение и предоставляет вспомогательные методы для common-преобразований.

### Методы

#### `as_str`

Возвращает внутреннее значение как срез строки.

**Возвращает:** `&str`

---

#### `as_list`

Разбирает значение как литерал списка AAM `[item, item, ...]` и возвращает элементы как `Vec<String>`. Возвращает `None`, если значение не в формате `[...]`.

**Возвращает:** `Option<Vec<String>>`

---

#### `as_object`

Разбирает значение как литерал встроенного объекта AAM `{ k = v, ... }` и возвращает `HashMap<String, String>` его полей. Возвращает `None`, если значение не может быть разобрано как встроенный объект.

**Возвращает:** `Option<HashMap<String, String>>`

---

## `AamlError`

Все ошибки, генерируемые конвейером разбора и валидации, реализуют `std::error::Error` и `Display`.

### Варианты

| Вариант | Описание |
|:---|:---|
| `IoError(io::Error)` | Произошла ошибка ввода-вывода при чтении файла. |
| `ParseError { line, content, details }` | Строка исходного кода не может быть разобрана как валидная инструкция AAML. |
| `NotFound(String)` | Имя ключа или типа не найдено в реестре или карте. |
| `InvalidValue(String)` | Значение не соответствует базовому ограничению типа (не схема). |
| `InvalidType { type_name, details }` | Значение не прошло валидацию против зарегистрированного или встроенного типа. |
| `DirectiveError(String, String)` | Директива столкнулась с ошибкой в своих аргументах. |
| `SchemaValidationError { ... }` | Нарушение ограничения схемы во время разбора или валидации. |

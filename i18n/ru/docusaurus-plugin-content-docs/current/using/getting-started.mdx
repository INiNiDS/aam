# Начало работы
Это краткое введение в AAM. Здесь рассматриваются основы и показывается, как создать простой двунаправленный маппинг между двумя переменными. Руководство содержит всё необходимое, чтобы начать использовать AAM в своём проекте.

## MVP
**Минимально рабочий продукт** (MVP) В AAM — это всего одна строка:
```aam
a = b
```
Синтаксис, создающий двунаправленный маппинг между `a` и `b`. По `a` можно получить `b`, а по `b` — `a`. Легко и просто
## Стандарты синтаксиса
AAM использует чистый минималистичный синтаксис, ориентированный на читаемость и простоту.

### Маппинг ключ-значение
По обе стороны от оператора `=` рекомендуется один пробел:
```aam
a = b
```

### Завершение выражений
Выражения завершаются **переносом строки** (`\n`). Точки с запятой или запятые в конце строки не нужны:
```aam
a = b
c = d
e = f
```
*Примечание*: Хотя запятые не используются для завершения выражений, они применяются как разделители элементов внутри специфических инлайн-структур, таких, как списки (```["Hello", "World"]```) и схемы в одну линию (```{name: string, id: i32}```).

### Комментарии
Однострочные комментарии используют префикс `#`:
```aam
# Это комментарий
a = b
```

### Директивы
Директивы (такие, как импорты и аналогичные объявления) используют префикс `@`:
```aam
@import other-file.aam
```

### Схемы в одну линию
Для однострочных схем в качестве разделителя используется запятая (`,`). Например:
```aam
@schema Player {name: string, id: i32}
```

### Многострочная схема
Для многострочных схем в качестве разделителя используется перенос строки (`\n`). Например:
```aam
@schema Player {
    name: string
    id: i32
}
```

### Соглашения об именовании

Для поддержания чистоты и согласованности проекта AAM использует особые соглашения об именовании:

* Переменные и маппинги: Используйте snake_case для всех имён переменных и маппингов.
    ```aam
    player_score = current_points
    ```

* Встроенные типы: Все примитивные и стандартные типы (например, i32, string или bool) пишутся в snake_case.

* Пользовательские типы и схемы: Используйте PascalCase для пользовательских типов и схем, чтобы чётко отличать их от встроенных примитивов.
    ```aam
    @schema PlayerData { ... } или @type UserProfile = string
    ```

# Схемы
В AAM схемы — это мощный способ определить структурированные данные. Они позволяют создавать сложные модели данных с чёткими типами и связями. Схемы можно использовать для различных целей: определения формы данных, валидации входных данных или генерации кода.
## Определение схемы
Для определения схемы в AAM используется директива `@schema`, за которой следует имя схемы и блок с полями и их типами. Например:
```aam
@schema Player {
    name: string
    id: i32
}
```
В этом примере определяется схема `Player` с двумя полями: `name` типа `string` и `id` типа `i32`. Схемы можно также определять инлайн, используя запятую в качестве разделителя:
```aam
@schema Player {name: string, id: i32}
```
Оба варианта эквивалентны и создают схему, которую можно использовать во всей конфигурации.
## Использование схем
После определения схемы вы можете использовать директиву `@derive` для создания экземпляров этой схемы. Например:
```aam
# file.aam
@schema Player {
    name: string
    id: i32
}
```
```aam
# main.aam
@derive file.aam::Player
name = "Alice"
id = 1
```
В этом примере мы наследуем схему `Player` из `file.aam` и затем задаём поля `name` и `id` для экземпляра схемы `Player`.
## Обязательность полей схемы
По умолчанию все поля схемы обязательны. Это означает, что при создании экземпляра схемы необходимо указать значения для всех полей. Если попытаться создать экземпляр без указания значений для всех обязательных полей, AAM выбросит ошибку. Например:
```aam
@derive file.aam::Player
name = "Alice"  # Это вызовет ошибку, так как 'id' обязательно, но не указано
```
Но если после имени поля добавить `*`, оно станет необязательным:
```aam
@schema Player {
    name: string
    id*: i32
}
```
В этом случае поле `id` необязателен, и можно создать экземпляр схемы `Player` без указания значения для `id`:
```aam
@derive file.aam::Player
name = "Alice"  # Это допустимо, так как 'id' необязателен
```
## Проверки
### Проверка типов
AAM автоматически проверяет типы полей при создании экземпляра схемы. Если попытаться присвоить значение, не соответствующее определённому типу, AAM выбросит ошибку. Например:
```aam
@derive file.aam::SchemaName
name = 123  # Ошибка: 'name' определено как string
id = "abc"  # Ошибка: 'id' определено как i32
```
### Проверка обязательных полей
AAM также проверяет, что все обязательные поля указаны при создании экземпляра схемы. Если пропустить обязательное поле, AAM выбросит ошибку. Например:
```aam
@derive file.aam::SchemaName
name = "Alice"  # Ошибка: 'id' обязательно, но не указано
```

@import ディレクティブ

@import ディレクティブはパーサーに外部の .aam ファイルを現在のドキュメントにマージするよう指示します。インポートされたファイルのすべてのキーと値のペア、型エイリアス、およびスキーマ定義が現在のスコープで利用可能になります。

@import path/to/other.aam

構文​

@import は単一の引用文字列（インポートするファイルへのパス）を取ります：

@import "shared/defaults.aam"
@import "../common/colors.aam"

パスはプロセスの作業ディレクトリではなく、ディレクティブを含むファイルに対して相対的に解決されます。

@import の動作​

パーサーが @import "file.aam" を検出したとき：

  1. パスを解決（現在のファイルのディレクトリに対して相対的）。
  2. 正規化（..、シンボリックリンクなどを解決）して、サイクル検出のための安定した識別子を得ます。
  3. 同じパーサーインスタンスと同じ visited セットを使用して、インポートされたファイルを再帰的にパースします。
  4. Child-Wins-on-Conflict セマンティクスを使用して現在のドキュメントにマージします。インポートする側のファイルにキーがすでに存在する場合、その値が優先されます。

マージのセマンティクス​

エンティティインポート時の動作
キーと値のペアマージされます。競合時は現在のファイルが優先されます。
@schema 定義スキーマレジストリに追加されます。
@type エイリアス型レジストリに追加されます。
入れ子の @import再帰的に解決されます。

実装例​

fn process_import(
    import_path: &str,
    current_file: &Path,
    current_doc: &mut AAML,
    visited: &mut HashSet<PathBuf>,
) -> Result<(), AamlError> {
    // 1. 現在のファイルのディレクトリからの相対パスを解決
    let raw_path = current_file
        .parent()
        .unwrap_or(Path::new("."))
        .join(import_path);

    // 2. 安定したアイデンティティのために正規化
    let canonical = raw_path.canonicalize().map_err(|_| {
        AamlError::FileNotFound { path: raw_path.clone() }
    })?;

    // 3. サイクル検出 — 下記セクションを参照
    if visited.contains(&canonical) {
        return Err(AamlError::CyclicImport { path: canonical });
    }

    // 4. インポートされたファイルを再帰的にパース
    visited.insert(canonical.clone());
    let imported_doc = parse_file_recursive(&canonical, visited)?;
    visited.remove(&canonical);

    // 5. マージ: 競合時は子（現在）が優先
    for (key, value) in imported_doc.elements {
        current_doc.elements.entry(key).or_insert(value);
    }
    for (name, schema) in imported_doc.schemas {
        current_doc.schemas.entry(name).or_insert(schema);
    }
    for (alias, ty) in imported_doc.type_registry {
        current_doc.type_registry.entry(alias).or_insert(ty);
    }

    Ok(())
}
注記

.entry(...).or_insert(...) の使用に注目してください。これにより、インポート元のファイルが常に優先されることが保証されます。インポートされたファイルの値は、現在のドキュメントにキーがまだ存在しない場合にのみ挿入されます。

循環依存の保護​

Anti-Loop — このセクションの実装は必須です

サイクル検出がないと、互いにインポートし合うファイルのペアによって無限再帰が発生し、スタックオーバーフローが引き起こされます。この保護はオプションではありません。

問題点​

次の2つのファイルを考えてみましょう：

# a.aam
@import "b.aam"
x = 1
# b.aam
@import "a.aam"
y = 2

保護がない場合、a.aam のパースが b.aam のパースをトリガーし、それが再び a.aam のパースをトリガーするという動作を無限に繰り返します。

解決策: "Visited Files" セット​

現在コールスタックにある正規化されたパスの HashSet<PathBuf>（または使用している言語の同等物）を保持します。インポートを処理する前に：

  1. 正規化されたパスがセットに既にあるか確認します。
  2. もしあれば — 直ちに CyclicImport エラーを返します。
  3. なければ — パスを追加し、再帰処理を行い、再帰から戻った後にパスを削除します。
fn parse_file_recursive(
    path: &Path,
    visited: &mut HashSet<PathBuf>,
) -> Result<AAML, AamlError> {
    let canonical = path.canonicalize()?;

    // ── サイクル保護 (Anti-loop guard) ──────────────────────────────────
    if visited.contains(&canonical) {
        return Err(AamlError::CyclicImport {
            path: canonical,
        });
    }
    visited.insert(canonical.clone());
    // ─────────────────────────────────────────────────────────────────────────

    let source = std::fs::read_to_string(&canonical)?;
    let doc = parse_source(&source, &canonical, visited)?;

    // 再帰の後に削除して、兄弟ファイルが引き続きインポートできるようにする
    visited.remove(&canonical);

    Ok(doc)
}

なぜ正規化されたパスなのか？​

生の文字列パスを使用するのは信頼性が低いためです：

  • ./shared/base.aam と shared/base.aam と ../project/shared/base.aam はすべて同じファイルを参照している可能性があります。
  • 正規化により、これらすべてが単一の絶対パスに解決され、安定したアイデンティティが得られます。

ダイヤモンドインポートは許可されます​

      main.aam
      /      \
  a.aam     b.aam
      \      /
      base.aam

base.aam は a.aam と b.aam の両方からインポートされます。これはサイクルではなく、別々のトラバーサルパスです。visited-set パターンは、最初のブランチが完了した後に base.aam がセットから削除されるため、2番目のブランチから再び入ることができ、正しく処理されます。

エラーレポート​

Error: cyclic import detected
  --> a.aam
   |
 1 | @import "b.aam"
   |
   = note: import chain: a.aam → b.aam → a.aam

HashSet と共に Vec<PathBuf> を渡してトラバーサルの順序を記録し、チェーンを再構築します。

キャッシュ​

1回の load() 呼び出し中に同じファイルを複数回パースするのは非効率的です。 visited セットとは別に、ドキュメントキャッシュ (HashMap<PathBuf, AAML>) を保持してください。

pub struct ParseContext {
    /// 現在コールスタックにあるファイル — サイクル検出用
    visited: HashSet<PathBuf>,

    /// すでにパース済みのドキュメント — 重複するI/Oとパースを回避
    cache: HashMap<PathBuf, AAML>,
}

キャッシュは正規化されたパスをキーにします。ファイルをパースする前に、まずキャッシュを確認してください。visited セットとキャッシュは目的が異なるため、混同しないでください。

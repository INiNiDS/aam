Директива @import

Директива @import предписывает парсеру объединить внешний файл .aam с текущим документом. Все пары ключ-значение, псевдонимы типов и определения схем из импортируемого файла становятся доступны в текущей области видимости.

@import path/to/other.aam

Синтаксис​

@import принимает одну строку в кавычках — путь к импортируемому файлу:

@import "shared/defaults.aam"
@import "../common/colors.aam"

Пути разрешаются относительно файла, содержащего директиву, а не относительно рабочей директории процесса.

Как это работает​

Когда парсер встречает @import "file.aam":

  1. Разрешает путь относительно директории текущего файла.
  2. Канонизирует его (разрешает .., симлинки и т.д.) для получения стабильного идентификатора для обнаружения циклов.
  3. Рекурсивно парсит импортированный файл, используя тот же экземпляр парсера и тот же набор посещенных файлов (visited).
  4. Объединяет результат с текущим документом, используя семантику Child-Wins-on-Conflict (побеждает дочерний файл) — если ключ уже существует в импортирующем файле, сохраняется его значение.

Семантика объединения​

СущностьПоведение при импорте
Записи ключ-значениеОбъединяются; текущий файл побеждает при конфликте
Определения @schemaДобавляются в реестр схем
Псевдонимы @typeДобавляются в реестр типов
Вложенные @importРазрешаются рекурсивно

Реализация​

fn process_import(
    import_path: &str,
    current_file: &Path,
    current_doc: &mut AAML,
    visited: &mut HashSet<PathBuf>,
) -> Result<(), AamlError> {
    // 1. Разрешение пути относительно директории текущего файла
    let raw_path = current_file
        .parent()
        .unwrap_or(Path::new("."))
        .join(import_path);

    // 2. Канонизация для стабильного идентификатора
    let canonical = raw_path.canonicalize().map_err(|_| {
        AamlError::FileNotFound { path: raw_path.clone() }
    })?;

    // 3. Обнаружение циклов — см. раздел ниже
    if visited.contains(&canonical) {
        return Err(AamlError::CyclicImport { path: canonical });
    }

    // 4. Рекурсивный парсинг импортированного файла
    visited.insert(canonical.clone());
    let imported_doc = parse_file_recursive(&canonical, visited)?;
    visited.remove(&canonical);

    // 5. Объединение: текущий (дочерний) файл побеждает при конфликте
    for (key, value) in imported_doc.elements {
        current_doc.elements.entry(key).or_insert(value);
    }
    for (name, schema) in imported_doc.schemas {
        current_doc.schemas.entry(name).or_insert(schema);
    }
    for (alias, ty) in imported_doc.type_registry {
        current_doc.type_registry.entry(alias).or_insert(ty);
    }

    Ok(())
}
примечание AAM

Обратите внимание на использование .entry(...).or_insert(...) — это гарантирует, что импортирующий файл всегда побеждает. Значения из импортированного файла вставляются только тогда, когда ключ еще не существует в текущем документе.

Защита от циклической зависимости​

Anti-Loop — этот раздел обязателен для реализации

Без обнаружения циклов пара файлов, которые импортируют друг друга, вызовет бесконечную рекурсию и переполнение стека. Эта защита не является опциональной.

Проблема​

Рассмотрим следующие два файла:

# a.aam
@import "b.aam"
x = 1
# b.aam
@import "a.aam"
y = 2

Без защиты парсинг a.aam вызвал бы парсинг b.aam, который снова вызвал бы парсинг a.aam, и так до бесконечности.

Решение: набор "посещенных файлов" (Visited Files)​

Поддерживайте HashSet<PathBuf> (или аналог в вашем языке) канонических путей, которые в данный момент находятся в стеке вызовов. Перед обработкой любого импорта:

  1. Проверьте, есть ли канонический путь уже в наборе.
  2. Если да — немедленно верните ошибку CyclicImport.
  3. Если нет — добавьте его, выполните рекурсию, а затем удалите его после возврата рекурсивного вызова.
fn parse_file_recursive(
    path: &Path,
    visited: &mut HashSet<PathBuf>,
) -> Result<AAML, AamlError> {
    let canonical = path.canonicalize()?;

    // ── Защита от циклов (Anti-loop guard) ──────────────────────────────────
    if visited.contains(&canonical) {
        return Err(AamlError::CyclicImport {
            path: canonical,
        });
    }
    visited.insert(canonical.clone());
    // ─────────────────────────────────────────────────────────────────────────

    let source = std::fs::read_to_string(&canonical)?;
    let doc = parse_source(&source, &canonical, visited)?;

    // Удаляем ПОСЛЕ рекурсии, чтобы сиблинги всё ещё могли импортироваться
    visited.remove(&canonical);

    Ok(doc)
}

Почему именно канонические пути?​

Использование обычных строковых путей ненадежно:

  • ./shared/base.aam и shared/base.aam и ../project/shared/base.aam могут ссылаться на один и тот же файл.
  • Канонизация разрешает всё это в один абсолютный путь, давая стабильный идентификатор.

Даймонд-импорты (ромбовидные) разрешены​

      main.aam
      /      \
  a.aam     b.aam
      \      /
      base.aam

base.aam импортируется как в a.aam, так и в b.aam. Это не цикл — это разные пути обхода. Паттерн с набором посещенных файлов обрабатывает это правильно, потому что base.aam удаляется из набора после завершения первой ветки и, следовательно, может быть снова посещен из второй ветки.

Сообщение об ошибке​

Error: cyclic import detected
  --> a.aam
   |
 1 | @import "b.aam"
   |
   = note: import chain: a.aam → b.aam → a.aam

Восстановите цепочку, передавая Vec<PathBuf> вместе с HashSet, чтобы записывать порядок обхода.

Кэширование​

Парсинг одного и того же файла несколько раз во время одного вызова load() неэффективен. Поддерживайте кэш документов (HashMap<PathBuf, AAML>), который отделен от набора visited:

pub struct ParseContext {
    /// Файлы, находящиеся в стеке вызовов — используются для обнаружения циклов.
    visited: HashSet<PathBuf>,

    /// Уже отпарсенные документы — позволяет избежать повторного ввода-вывода и парсинга.
    cache: HashMap<PathBuf, AAML>,
}

Кэш индексируется каноническим путем. Перед парсингом файла сначала проверьте кэш. Набор посещенных файлов и кэш служат разным целям и не должны смешиваться.
